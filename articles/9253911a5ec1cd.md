## 概要

前回、GitBucket の [Ace](https://ace.c9.io/)(Ajax.org Cloud9 Editor) で vim キーバインドなどが使えるようにして Wiki でも利用できるよう改造し、プルリクエストがマージされました。

しかしながら、Wiki で使えていた絵文字の入力支援が使えなくなりました。

今回、Markdown の編集においては、絵文字の入力を補完できるように修正しました。併せて、他のファイル形式では、Ace の入力補完機能をフルで使用できるように修正しました。

<https://qiita.com/yasumichi/items/62b5362b0371aa958292>

## 最初のプルリクエスト

2026/02/14 に最初のプルリクエストを発行しました。

- [Add text completion by yasumichi · Pull Request #3964 · gitbucket/gitbucket](https://github.com/gitbucket/gitbucket/pull/3964)

しかしながら、実装方法について、以下のご指摘を受けました。

> I wonder if it's possible to apply GitBucket's [suggestion provider plugins](https://github.com/gitbucket/gitbucket/blob/f72617769bd555a5fed699c35babfc17d840ba56/src/main/scala/gitbucket/core/plugin/SuggestionProvider.scala) to Ace editor, rather than implementing specific text completion in GitBucket core side.
>
> Actually, text completion for emoji characters is provided by gitbucket-emoji-plugin. It would be great if we can reuse it.
> <https://github.com/gitbucket/gitbucket-emoji-plugin/blob/57c72cb1728e0112f825337882cbd90a8dfe7a9e/src/main/scala/Plugin.scala#L32-L42>

GitBucket には、入力補完の候補をプラグインで提供できる仕組みを [src/main/scala/gitbucket/core/plugin/SuggestionProvider.scala](https://github.com/gitbucket/gitbucket/blob/f72617769bd555a5fed699c35babfc17d840ba56/src/main/scala/gitbucket/core/plugin/SuggestionProvider.scala) で提供しているようです。

GitBucket に同梱されている [gitbucket-emoji-plugin](https://github.com/gitbucket/gitbucket-emoji-plugin) は、この仕組みを利用しており、絵文字の入力候補を提供しているそうです。

最初のプルリクエストでは、絵文字の入力候補を[独自に準備していた](https://github.com/gitbucket/gitbucket/pull/3964/changes#diff-21ef24cc14939e3cd368549806592f0ced23b814eeaeb7aeadde3ad48a46c0d7)ため、gitbucket-emoji-plugin の入力候補を再利用した方が良いとのことでした。

### SuggestionProvider から Completer を展開するための Twirl テンプレートを追加

[src\main\twirl\gitbucket\core\helper\attached.scala.html](https://github.com/gitbucket/gitbucket/blob/35d670843f4fde4fb2c52071ed1f1638531de9ea/src/main/twirl/gitbucket/core/helper/attached.scala.html) を参考に [src/main/twirl/gitbucket/core/helper/completion.scala.html](https://github.com/gitbucket/gitbucket/pull/3974/changes#diff-15fd2b46376cadf7d9f4c6e1fd22d04326d1d9536009b919f4dff04c28a53f7c) を新規作成しました。

後半の `<script>` 要素内で実際の入力補完機能を実装しています。

```scala
<script>
  var mdCompleters = [];

  @gitbucket.core.plugin.PluginRegistry().getSuggestionProviders.map { provider =>
    @if(provider.context.contains(completionContext)){
      var @provider.id = @Html(helpers.json(provider.options(repository).map { case (value, label) =>
        Map("value" -> (s"${provider.prefix}${value}${provider.suffix}"), "label" -> label)
      }));

      var @{provider.id}Completer = {
        triggerCharacters: [ `@provider.prefix.charAt(0)` ],
        getCompletions: function (editor, session, pos, prefix, callback) {
          if (session.$mode && session.$mode.$id === 'ace/mode/markdown') {
            callback(null, @{provider.id}.map(function (table) {
              var token = session.getTokenAt(pos.row, pos.column).value.slice(-1);
              return {
                caption: table.description,
                value: token == "@provider.prefix.charAt(0)" ? table.value.replace(token, "") : table.value,
                className: "@{provider.id}" == "emoji" ? table.label.replace(/[^a-z0-9\-_]/g, '_') : "@{provider.id}",
                meta: "@{provider.id}"
              };
            }));
          }
        },
        id: "@{provider.id}Completer"
      };

      mdCompleters.push(@{provider.id}Completer);

      @Html(provider.additionalScript(repository))
    }
  }
</script>
```

このループでは、 PluginRegistry から SuggestionProvider のシーケンスを取得し、各要素を provider に代入し、順に処理していきます。

取り出した provider が、入力補完用の completionContext (completion.scala.html を import する際に "wiki" にしています。)を含んでいれば、以下の処理を行います。


provider が提供する入力候補のリストを生成します。絵文字であれば、以下のように展開されます。

```javascript

```

次が、Completer の本体です。

```Scala
      var @{provider.id}Completer = {
        triggerCharacters: [ `@provider.prefix.charAt(0)` ],

          if (session.$mode && session.$mode.$id === 'ace/mode/markdown') {
```

絵文字であれば、以下のように展開されます。

```javascript
      var emojiCompleter = {
        triggerCharacters: [ `:` ],
        getCompletions: function (editor, session, pos, prefix, callback) {
          if (session.$mode && session.$mode.$id === 'ace/mode/markdown') {
            callback(null, emoji.map(function (table) {
              var token = session.getTokenAt(pos.row, pos.column).value.slice(-1);
              return {
                caption: table.description,
                value: token == ":" ? table.value.replace(token, "") : table.value,
                className: "emoji" == "emoji" ? table.label.replace(/[^a-z0-9\-_]/g, '_') : "emoji",
                meta: "emoji"
              };
            }));
          }
        },
        id: "emojiCompleter"
      };
```

これは、[Ace](https://ace.c9.io/)(Ajax.org Cloud9 Editor) の [ace-internal.d.ts](https://github.com/ajaxorg/ace/blob/05fbd6a33e334586479a5a057fca28aaee11c825/ace-internal.d.ts#L981) で定義されている `interface Completer` のメンバーを一部定義しています。なお、

```javascript
                value: token == ":" ? table.value.replace(token, "") : table.value,
```

としているのは、 `triggerCharacters` である `:` が入力された状態で候補を決定すると `:+1:` のようになるべきところが、`::+1:` のように冒頭の `:` が重複してしまうため、それを避けるための苦肉の策です。

AI は、`insertMatch` というメンバー関数を定義してそこで処理するように提案してきたのですが、一向に呼ばれる気配がありません。調査した結果、

- [Autocomplete](https://github.com/ajaxorg/ace/blob/05fbd6a33e334586479a5a057fca28aaee11c825/src/autocomplete.js#L470) クラスでは、CompletionProvider の insertMatch を呼ぶのであって、Completer.insertMatch を呼ぶわけではない。 ~~各 Completer の insertMatch を呼んでくれる。~~
- [InlineAutocomplete](https://github.com/ajaxorg/ace/blob/05fbd6a33e334586479a5a057fca28aaee11c825/src/ext/inline_autocomplete.js#L114C51-L114C64) クラスでは、呼んでくれない。~~(insertByIndex を定義すれば良いのかも。後日、研究。)~~

という結論に至りました。

```javascript
      mdCompleters.push(@{provider.id}Completer);
```

先ほど定義した Completer を `mdCompleters` に追加しています。

```Scala
      @Html(provider.additionalScript(repository))
```

provider が追加スクリプトを提供していれば、それを展開します。(現状では、対応しているプロバイダーが存在しないため、出力されませんが…)

### Ace を制御するスクリプトの修正

[src/main/webapp/assets/common/js/editor.js](https://github.com/gitbucket/gitbucket/pull/3974/changes#diff-bbfd8140da7a1ff77a3da092ba0cf68508e97da7f8acdaf7a252d924c293c0ef) を修正しました。

```javascript
var originalCompleter = null;
```

デフォルトの Completer リストを保存するための `originalCompleter` という変数を定義しています。

```javascript
        /**
         * A function to update the file mode of the Ace editor
         */
        const updateFileMode = function () {
            if (gitbucket.getFileName() != "") {
                var modelist = ace.require("ace/ext/modelist");
                var mode = modelist.getModeForPath(gitbucket.getFileName());
                gitbucket.editor.getSession().setMode(mode.mode);
                if (mode.name == "markdown") {
                    markdownToolBar.style.display = "block";
                    gitbucket.editor.completers = mdCompleters ?? [];
                } else {
                    markdownToolBar.style.display = "none";
                    gitbucket.editor.completers = originalCompleter ?? [];
                }
            } else {
                    markdownToolBar.style.display = "none";
                    gitbucket.editor.completers = originalCompleter ?? [];
            }
        };
```

markdown モードの際は、completion.scala.html で定義した `mdCompleters` を入力候補とし、それ以外のモードでは、`originalCompleter` を使用するように修正しました。

```javascript
const langtools = ace.require("ace/ext/language_tools");
```

入力補完機能を有効にするため、Ace の language_tools という拡張を読み込みます。

```javascript
        // enable auto completion
        gitbucket.editor.setOptions({
            enableBasicAutocompletion: true,
            enableSnippets: true,
            enableLiveAutocompletion: true
        });
        originalCompleter = gitbucket.editor.completers;
```

Ace のオプションを設定し、入力補完機能を有効にするとともにデフォルトの Completer リストを保存しています。
